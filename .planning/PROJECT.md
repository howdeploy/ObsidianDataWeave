# ObsidianDataWeave

## What This Is

Полный тулкит для автоматического превращения сырых ресерчей из NotebookLM/Google Drive в структурированную базу знаний Obsidian. Claude Code скилл берёт документ с Google Drive, разбивает его на атомарные заметки по методологии MOC + Zettelkasten, расставляет теги и связи, и сохраняет в Obsidian волт. Публикуется как open-source решение на GitHub с установкой в одну команду.

## Core Value

Одна команда превращает любой ресерч-документ в набор правильно структурированных, связанных между собой атомарных заметок внутри Obsidian — без ручной работы.

### Out of Scope

- Плагин для Obsidian (это скиллы Claude Code, не JS плагин) — отдельная экосистема
- Обратная синхронизация Obsidian → NotebookLM — однонаправленный поток
- Автоматический мониторинг Google Drive (пока ручной запуск по команде) — усложнение v1
- Редактирование существующих заметок (только создание новых) — риск потери данных

## Context

- Obsidian волт: `/mnt/sda1/KISA's Space/` (папки: "Гайды, разборы", "Ресерч и исследования")
- rclone уже настроен (remote: `gdrive:`), скилл синхронизации работает
- Smart Connections плагин установлен в Obsidian (векторные эмбеддинги)
- Два .docx файла в проекте — ресерчи по архитектуре заметок, используются как правила:
  - `Архитектура Второго мозга: Синхронизация Obsidian и Claude MCP.docx`
  - `Smart Connections: Интеллектуальный мозг вашей базы Obsidian.docx`
- Пользователь делает ресерчи в NotebookLM (саммари из источников + диалоги с AI), экспортирует на Drive
- Текущие заметки: в основном дип-ресерчи на разные темы, без MOC-системы
- Цель: начать вести все новые заметки по MOC + Zettelkasten

## Constraints

- **Платформа**: Linux (Manjaro), Claude Code CLI — скиллы должны быть bash/python совместимы
- **Зависимости**: rclone (уже есть), python3 (для парсинга docx), pandoc (опционально)
- **Obsidian совместимость**: стандартный markdown, [[wikilinks]], YAML frontmatter для тегов
- **Установка**: одна команда (`bash install.sh` или подобное), минимум интерактива
- **GitHub**: публичный репозиторий, понятный README на русском и английском
